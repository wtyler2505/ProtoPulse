The backlog uses a structured epic format with goal, scope, dependencies, risks, exit criteria, and verification sections. The trigger for decomposition is communication: when a single row with ID/status/complexity cannot convey enough context for another engineer to plan or implement the work. Six epics (A-F) demonstrate this pattern for TinkerCAD parity, cross-tool integration, manufacturing trust, and UX spectrum.
