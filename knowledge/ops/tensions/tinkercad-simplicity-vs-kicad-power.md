summary: The "one obvious start flow" goal for beginners conflicts with advanced users needing full parameter control
type: tension
created: 2026-03-13
---

TinkerCAD's appeal is zero-learning-curve simulation. KiCad's appeal is professional-grade output. ProtoPulse tries to be both. Every surface where this tension manifests — simulation start (auto-detect vs manual), component placement (drag-drop vs parametric), export (one-click vs configuration) — forces a choice about which audience to serve first. Progressive disclosure is the theoretical answer but adds implementation complexity at every surface.
