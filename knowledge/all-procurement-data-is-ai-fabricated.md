# All procurement data in the BOM is AI-simulated with zero real supplier API connections

The product analysis flagged this as one of the top three critical issues: pricing, stock levels, and lead times shown in the BOM are entirely fabricated by the AI. A user making purchasing decisions — comparing suppliers, budgeting a project, ordering parts — would get wrong prices, wrong availability, and wrong lead times. The data looks authoritative in the UI but has no connection to reality.

This is worse than showing no data at all, because it creates false confidence. A user who sees "$0.15 per capacitor, 45,000 in stock" may proceed to order only to discover the part is discontinued or costs $2.50. The fix is straightforward (integrate real supplier APIs from DigiKey, Mouser, LCSC, or JLCPCB), and the report promoted this from P2 to P1 based on cross-phase evidence. Supplier API integration has since been implemented in Wave 36, but the lesson persists: simulated data that looks real is more dangerous than missing data.
