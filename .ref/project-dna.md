## Architecture in 30 Seconds

All-in-one browser-based EDA suite for makers, hobbyists, and anyone working with microcontrollers, electronics, robotics, or embedded systems. Born from the OmniTrek Nexus rover project where no single tool covered the full journey from "I don't know electronics" to "here are my Gerbers." Not project-specific — ProtoPulse is the universal tool. Think TinkerCad + Fritzing + KiCad + Arduino IDE unified with AI.

Monolithic Express server serves both the API and the Vite-built SPA. AI chat supports 134 AI tools that directly manipulate project state (add nodes, generate architectures, manage BOM, run DRC, circuit operations, exports). Multi-format export: KiCad, Eagle, SPICE, Gerber, drill, pick-and-place, design report, FMEA, firmware scaffold, PDF. All state flows through a single `ProjectProvider` React context backed by React Query mutations.

## Onboarding Path (read in this order)

1. `shared/schema.ts` — 27 Drizzle tables + Zod insert schemas; defines every data shape
2. `shared/component-types.ts` — component editor type system (shapes, connectors, DRC rules)
3. `shared/drc-engine.ts` — design rule checking engine (shared between server + client)
4. `server/storage.ts` — `IStorage` interface + `DatabaseStorage` with LRU cache
5. `server/routes.ts` — main REST endpoints; maps URLs to storage + AI calls
6. `server/circuit-routes.ts` — circuit schematic API endpoints
7. `server/ai.ts` — AI prompt construction, 134 AI tools, streaming SSE responses, multi-model routing
8. `client/src/lib/project-context.tsx` — monolithic context with 40+ state values + mutations
9. `client/src/pages/ProjectWorkspace.tsx` — 3-panel layout wiring everything together

## Non-Obvious Conventions

- Soft deletes on `projects`, `architecture_nodes`, `architecture_edges`, `bom_items` — always filter `isNull(deletedAt)`
- `PROJECT_ID = 1` is hardcoded in `project-context.tsx` — blocks multi-project (known debt)
- Auth uses `X-Session-Id` header, not cookies
- Node IDs use `crypto.randomUUID()`, not `Date.now()`
- BOM `totalPrice` is computed server-side from `quantity * unitPrice`
- AI system prompt rebuilds full project state on every request — O(N) sequential queries
- Cache uses LRU eviction. Invalidation uses prefix matching (invalidating `"nodes:1"` clears all `"nodes:1*"` keys)
- `@/*` path alias maps to `client/src/*`, `@shared/*` maps to `shared/`
- ESLint enforces `import type` for type-only imports
- Response envelope inconsistency: DELETEs return 204 in routes.ts but `{ success: true }` in circuit-routes.ts

## Architectural Decisions

- **Express 5 (not Next.js)**: SPA with separate API; no SSR needed for a tool app
- **React Query over Redux**: Server state only, no client-only global state needed
- **Dual AI providers**: Claude for complex actions, Gemini as fallback/alternative; multi-model routing
- **Monolithic context**: `ProjectProvider` is known tech debt — works for single project
- **shadcn/ui**: Copy-paste components, full control, consistent dark theme
- **Vitest over Jest**: Native Vite integration, workspace projects for server/client separation
- **LRU cache**: Custom implementation replacing original FIFO Map-based approach

## Gotchas

- Deprecated endpoints `/api/bom/:id` and `/api/validation/:id` still exist — prefer `/api/projects/:id/bom/:bomId`
- Originally a Replit project — some Replit-specific env vars remain (`REPL_ID`, `REPLIT_DEV_DOMAIN`)
- `server/__tests__/api.test.ts` uses `node:test` runner (not Vitest) — excluded from `npm test`
- No API versioning — all routes at `/api/*` with no version prefix
