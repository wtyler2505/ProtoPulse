## Tech Stack
- **Runtime**: Node.js (Express 5, tsx for dev)
- **Framework**: React 19 + Vite 7 (SPA, no SSR)
- **Language**: TypeScript 5.6
- **Styling**: Tailwind v4 + shadcn/ui (New York dark theme)
- **Key Dependencies**:
  - `@xyflow/react` — interactive architecture block diagrams
  - `drizzle-orm` + `pg` — PostgreSQL ORM with Zod schema integration
  - `@tanstack/react-query` — server state management (no Redux/Zustand)
  - `@anthropic-ai/sdk` + `@google/generative-ai` — dual AI provider (Claude + Gemini)
  - `wouter` — lightweight client routing
  - `jszip` — FZPZ component import/export

## Architecture in 30 Seconds
Browser-based EDA (Electronic Design Automation) tool for architecture block diagrams, BOM management, design validation, and AI-assisted design. Monolithic Express server serves both the API and the Vite-built SPA. AI chat supports 53 action types that directly manipulate project state (add nodes, generate architectures, manage BOM, run DRC). All state flows through a single `ProjectProvider` React context backed by React Query mutations.

## Non-Obvious Conventions
- Soft deletes on `projects`, `architecture_nodes`, `architecture_edges`, `bom_items` — always filter `isNull(deletedAt)`
- `PROJECT_ID = 1` is hardcoded in `project-context.tsx` — blocks multi-project (known debt)
- Auth uses `X-Session-Id` header, not cookies
- Node IDs use `crypto.randomUUID()`, not `Date.now()`
- BOM `totalPrice` is computed server-side from `quantity * unitPrice`
- AI system prompt rebuilds full project state on every request
- Cache invalidation uses prefix matching (invalidating `"nodes:1"` clears all `"nodes:1*"` keys)
- `@shared/*` path alias maps to `shared/` directory (configured in tsconfig + vite)

## Architectural Decisions
- **Express 5 (not Next.js)**: SPA with separate API; no SSR needed for a tool app
- **React Query over Redux**: Server state only, no client-only global state needed
- **Dual AI providers**: Claude for complex actions, Gemini as fallback/alternative
- **Monolithic context**: `ProjectProvider` is known tech debt — works for single project
- **shadcn/ui**: Copy-paste components, full control, consistent dark theme

## Gotchas
- Deprecated endpoints `/api/bom/:id` and `/api/validation/:id` still exist — prefer `/api/projects/:id/bom/:bomId`
