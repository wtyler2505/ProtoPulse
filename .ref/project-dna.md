  - `@tanstack/react-query` ‚Äî server state management (no Redux/Zustand)
  - `@anthropic-ai/sdk` + `@google/genai` ‚Äî dual AI provider (Claude + Gemini)

  - `jszip` ‚Äî FZPZ component import/export
  - `helmet` + `express-rate-limit` ‚Äî security middleware

## Architecture in 30 Seconds

Browser-based EDA (Electronic Design Automation) tool for architecture block diagrams, circuit schematic editing, BOM management, design validation (DRC/ERC), and AI-assisted design. Monolithic Express server serves both the API and the Vite-built SPA. AI chat supports 82 AI tools that directly manipulate project state (add nodes, generate architectures, manage BOM, run DRC, circuit operations, exports). Multi-format export: KiCad, Eagle, SPICE, Gerber, drill, pick-and-place, design report, FMEA, firmware scaffold, PDF. All state flows through a single `ProjectProvider` React context backed by React Query mutations.

## Non-Obvious Conventions

- Soft deletes on `projects`, `architecture_nodes`, `architecture_edges`, `bom_items` ‚Äî always filter `isNull(deletedAt)`
- `PROJECT_ID = 1` is hardcoded in `project-context.tsx` ‚Äî blocks multi-project (known debt)
- Auth uses `X-Session-Id` header, not cookies
- Node IDs use `crypto.randomUUID()`, not `Date.now()`
- BOM `totalPrice` is computed server-side from `quantity * unitPrice`
- AI system prompt rebuilds full project state on every request ‚Äî O(N) sequential queries
- Cache uses LRU eviction. Invalidation uses prefix matching (invalidating `"nodes:1"` clears all `"nodes:1*"` keys)
- `@/*` path alias maps to `client/src/*`, `@shared/*` maps to `shared/`
- ESLint enforces `import type` for type-only imports
- Response envelope inconsistency: DELETEs return 204 in routes.ts but `{ success: true }` in circuit-routes.ts

## Architectural Decisions

- **Express 5 (not Next.js)**: SPA with separate API; no SSR needed for a tool app
- **React Query over Redux**: Server state only, no client-only global state needed
- **Dual AI providers**: Claude for complex actions, Gemini as fallback/alternative; multi-model routing
- **Monolithic context**: `ProjectProvider` is known tech debt ‚Äî works for single project
- **shadcn/ui**: Copy-paste components, full control, consistent dark theme
- **Vitest over Jest**: Native Vite integration, workspace projects for server/client separation
- **LRU cache**: Custom implementation replacing original FIFO Map-based approach

## Gotchas

- Deprecated endpoints `/api/bom/:id` and `/api/validation/:id` still exist ‚Äî prefer `/api/projects/:id/bom/:bomId`
- Originally a Replit project ‚Äî some Replit-specific env vars remain (`REPL_ID`, `REPLIT_DEV_DOMAIN`)
- `server/__tests__/api.test.ts` uses `node:test` runner (not Vitest) ‚Äî excluded from `npm test`
- No API versioning ‚Äî all routes at `/api/*` with no version prefix
