# OmniTrek Nexus Wiring Diagram: Batteries and Power System

## Document Overview

This document provides comprehensive wiring instructions for the battery pack and power distribution system of the OmniTrek Nexus 4WD rover. This covers battery configuration, voltage regulation, power distribution to motor controllers, MCU power supply, and critical safety systems.

## Power System Architecture

### System Power Requirements

**Motor Controller Power:**
- Quantity: 4 Г— RioRand 350W BLDC Controllers
- Nominal voltage: 36V DC
- Continuous current per controller: 10A
- Peak current per controller: 20A (short duration)
- Total continuous power: 1,400W (4 Г— 350W)
- Total peak power: 2,000W+

**Microcontroller Power:**
- NodeMCU 32S: 5V USB or 3.3V direct
- Current consumption: 200-500mA (WiFi active)
- Peak current: 1A (WiFi transmit bursts)

**Auxiliary Systems:**
- Hall sensors: 4 Г— 60mA = 240mA @ 5V
- Status LEDs and indicators: ~100mA @ 5V
- Cooling fans (if equipped): 500mA @ 12V

## Battery Configuration

### Primary Battery Pack (36V Configuration)

**Option A: 10S Li-ion Pack**
- Cell configuration: 10S (10 cells in series)
- Nominal voltage: 36V (3.6V Г— 10)
- Fully charged: 42V (4.2V Г— 10)
- Discharge cutoff: 30V (3.0V Г— 10)
- Recommended capacity: 10-20Ah
- Chemistry: Li-ion 18650 or 21700 cells

**Option B: 3Г— 12V Lead Acid**
- Battery type: 12V SLA or AGM
- Configuration: 3 batteries in series
- Nominal voltage: 36V (12V Г— 3)
- Fully charged: ~39V
- Discharge cutoff: ~31.5V (10.5V Г— 3)
- Recommended capacity: 7-12Ah each

**Option C: 10S LiFePO4 Pack**
- Cell configuration: 10S
- Nominal voltage: 32V (3.2V Г— 10)
- Fully charged: 36.5V (3.65V Г— 10)
- Discharge cutoff: 25V (2.5V Г— 10)
- More stable chemistry, longer life
- Note: Slightly lower voltage than Li-ion

## Verification Checklist

### Pre-Power Tests

- [ ] All battery voltages verified individually
- [ ] Series connections verified (36V total)
- [ ] Polarity marked clearly at all points
- [ ] All fuses installed and correct rating
- [ ] E-stop circuit tested (disconnects power)
- [ ] BMS balanced and functional (if Li-ion)
- [ ] No short circuits (resistance test all rails)

### Power-On Sequence

1. [ ] E-stop engaged (pressed/open)
2. [ ] Main switch OFF
3. [ ] Connect battery pack
4. [ ] Verify voltage at distribution (should be 0V with E-stop open)
5. [ ] Release E-stop
6. [ ] Verify voltage at distribution (~36V)
7. [ ] Turn on main switch
8. [ ] Verify voltage at each motor controller input
9. [ ] Verify 12V rail (if DC-DC converter installed)
10. [ ] Verify 5V rail (for NodeMCU power)
11. [ ] Power on NodeMCU and verify boot

### Load Testing

- [ ] Each motor controller powers motor individually
