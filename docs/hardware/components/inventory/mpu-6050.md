## Specifications

### Technical Specifications

- **Model/Part Number**: MPU-6050
- **Manufacturer**: InvenSense (TDK)
- **Technology**: MEMS 6-axis motion tracking
- **Form Factor**: 20 x 15 mm

### Electrical Characteristics

- **Operating Voltage**: 3.3V-5V
- **Current Consumption**: 3.6mA typical, 500µA standby
- **Power Requirements**: 3.3V or 5V supply
- **Logic Levels**: 3.3V logic levels

### Performance Characteristics

- **Accelerometer Range**: ±2, ±4, ±8, ±16 g (16-bit resolution)
- **Gyroscope Range**: ±250, ±500, ±1000, ±2000 °/s (16-bit resolution)
- **Interface Speed**: I2C up to 400 kHz
- **Accuracy/Precision**: High-precision motion tracking

### Physical Characteristics

- **Dimensions**: 20 x 15 mm
- **Weight**: 2 g
- **Operating Temperature**: -40 to 85°C
- **Mounting**: PCB mounting holes

## Troubleshooting

### Common Issues

| Symptom           | Possible Cause       | Solution                        |
| ----------------- | -------------------- | ------------------------------- |
| No sensor data    | I2C connection issue | Check wiring, pull-up resistors |
| Excessive drift   | Improper calibration | Run calibration routine         |
| Noisy readings    | Power supply noise   | Add decoupling capacitor        |
| Wrong I2C address | Address conflict     | Use alternate address pins      |

### Debugging Steps

1. Verify I2C connections (SCL, SDA, VCC, GND)
2. Check I2C address with I2C scanner
3. Test with minimal example sketch
4. Verify power supply stability

### Performance Issues

- **Drift**: Temperature changes, improper calibration
- **Noise**: Power supply issues, electromagnetic interference
- **Complete Failure**: Physical damage, I2C bus issues

## Project Usage

### Current Implementation

- **Role in Project**: Provides orientation data for rover stability and navigation
- **Integration Status**: Connected to Arduino Mega via I2C
- **Performance in Project**: Stable motion tracking with minimal drift
- **Known Limitations**: Requires periodic calibration, temperature sensitivity

### Code Examples

```cpp
#include <Adafruit_MPU6050.h>
#include <Adafruit_Sensor.h>
#include <Wire.h>

Adafruit_MPU6050 mpu;

void setup() {
  Serial.begin(9600);

  if (!mpu.begin()) {
    Serial.println("Failed to find MPU6050 chip");
    while (1);
  }

  mpu.setAccelerometerRange(MPU6050_RANGE_8_G);
  mpu.setGyroRange(MPU6050_RANGE_500_DPS);
  mpu.setFilterBandwidth(MPU6050_BAND_21_HZ);
}

void loop() {
  sensors_event_t a, g, temp;
  mpu.getEvent(&a, &g, &temp);

  Serial.print("Accel X: "); Serial.print(a.acceleration.x);
  Serial.print(" Y: "); Serial.print(a.acceleration.y);
  Serial.print(" Z: "); Serial.println(a.acceleration.z);

  Serial.print("Gyro X: "); Serial.print(g.gyro.x);
  Serial.print(" Y: "); Serial.print(g.gyro.y);
  Serial.print(" Z: "); Serial.println(g.gyro.z);

  delay(100);
}
```

### Configuration

- **Default Settings**: ±2g accelerometer, ±250°/s gyroscope
- **Optimal Settings**: ±8g accelerometer, ±500°/s gyroscope for rover
- **Configuration Files**: Arduino sketch settings
