THE THREE SPACES (memorize this — it's the architectural axiom)
1. **`inbox/`** — Raw material to process. Articles, voice dumps, URLs, imported sources, NotebookLM Studio outputs. Files here have NO claim of correctness; they're queue items.
2. **`knowledge/`** — Atomic, evidenced claims. Each note = one claim, kebab-case slug as title-claim, frontmatter includes provenance + reviewed date.
3. **`ops/`** — Operational state about the system itself. `methodology/` (vault self-knowledge), `derivation.md` (why this configuration), `observations/` (friction signals), `index/` (backlinks), `queue/` (gap-stubs, processing tasks), `sessions/` (logs), `health/` (reports).

The boundary discipline: NEVER write directly to `knowledge/`. inbox/ → `/extract` → knowledge/. NEVER pollute `ops/` with claims; observations and methodology only. The boundary is the methodology.

THE PIPELINE (sequential discipline)
1. **`/seed <file>`** — Add a source to inbox queue + create extract task.
2. **`/extract <inbox-file>`** — Mine atomic claims into `knowledge/`. Comprehensive extraction default — every insight that serves the domain gets extracted. Skip rate <10% for domain-relevant sources.
3. **`/connect <note>`** — Find connections between notes; update MOCs. Semantic judgment required.
4. **`/revisit <note>`** — Update old notes with new connections (the backward pass `/connect` doesn't do).
5. **`/verify <note>`** — Recite (description quality cold-read prediction) + validate (schema compliance) + review (health checks).
6. **`/pipeline <file>`** — End-to-end orchestration of the above.

QUALITY GATES (enforced by skills + hooks)

- **`vault-quality-gate`:** Pre-commit gate. Notes failing land in `inbox/review/` with a triage marker.

PROVENANCE TAXONOMY (use these terms, don't substitute)

- `vendor-doc` — manufacturer app note, white paper, design guide.
- `textbook` — published book.
- `paper` — peer-reviewed academic paper.
- `experiment` — Tyler's bench measurement (link to pp-bench observation).

- `other` — escape hatch; describe in body.
