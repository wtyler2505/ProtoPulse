KEY FACTS

**1088AS 8x8 dot-matrix common-cathode pinout TRAP:**
- 16-pin DIP (8 row + 8 col).
- **Pin numbering is NON-SEQUENTIAL across rows and columns.** Datasheet pins do NOT correspond to physical row/col order in any obvious way.
- Orientation verification is MANDATORY before placement: look for the pin-1 indicator, cross-check against the datasheet diagram for which physical pin = which row/col.
- The breadboard editor's exact-part flow encodes the verified pin map; manual placement bypasses that protection.

**Common-cathode (CC) vs common-anode (CA) — silent firmware break:**
- Same package, same pinout, electrically OPPOSITE polarity.
- Swap one for the other → segments don't light, firmware looks correct, hours lost debugging.
- Verify the part designation (1088AS = CC; 1088BS = CA) physically before wiring.
- MAX7219 ONLY drives CC (DIG pins sink current to ground via internal switches). CA displays + MAX7219 = no light, no error, debugging hell.

**Direct-drive GPIO cost (the multiplex tax):**
- Single-digit 7-segment direct-driven: 8 GPIO (7 segments + 1 dp) + 1 common = 9 pins for ONE digit.
- 4-digit 7-segment direct-driven (HS420561K): 12 GPIO (7 seg + 1 dp + 4 digit-commons), CPU multiplexing at >100Hz to avoid flicker = `direct-driving-a-4-digit-7-segment-display-consumes-12-gpio-pins-and-requires-constant-software-multiplexing`.
- 8x8 matrix (1088AS) direct-driven: 16 GPIO + CPU lock to refresh = `direct-driving-an-8x8-led-matrix-consumes-16-io-pins-and-locks-the-cpu-to-display-refresh`.
- ATmega328P (Arduino UNO, 14 GPIO) cannot direct-drive an 8x8 — period.

**MAX7219 driver advantage:**
- 3-wire SPI (DIN, CS, CLK) → controls 8 digits OR 8x8 matrix.
- Includes BCD decoder, brightness PWM, scan-limit register.
- Cascadable for larger displays (DOUT of one MAX7219 → DIN of next).
- ONLY drives common-cathode (`max7219-only-works-with-common-cathode-displays-because-dig-pins-are-current-sinks`).
- Daisy-chain idiom: serial protocol with 16-bit "register | data" frames, latch on CS rising edge.

**Per-segment current:**
- 1088AS LEDs: 20mA per segment typical, 30mA peak.
- 4-digit 7-segment: 10-20mA per segment with multiplex (1/4 duty cycle requires higher peak current).
- MAX7219 internal current limit set via Iset resistor (typically 30kΩ → ~30mA per segment peak).

**Refresh discipline:**
- Direct-drive: ≥60Hz refresh per digit/row to avoid visible flicker (≥240Hz for a 4-digit). Each digit on 1/4 the time → peak segment current must be 4× average for desired brightness.
- MAX7219: handles refresh internally at ~800Hz; firmware just writes character data.

**Common alternatives:**
- HT16K33 — I²C driver, similar capability to MAX7219, different polarity support.
- TM1637 / TM1638 — single-package CC 7-segment + button modules.
- SHIELD-style 8x8 matrices with built-in MAX7219 (the LW-45-24P unidentified module is one).

ANTI-PATTERNS (NEVER do these)
- Never recommend wiring a 1088AS by guessing pin order. Always verify via datasheet + breadboard-editor exact-part lookup.
- Never recommend MAX7219 + common-anode display.
- Never recommend direct-driving an 8x8 matrix from a 14-GPIO MCU.
- Never substitute 1088AS (CC) with 1088BS (CA) without firmware-side polarity flip.
- Never skip the segment-current-limit resistor (typically 220-330Ω per segment for a direct-drive design).
