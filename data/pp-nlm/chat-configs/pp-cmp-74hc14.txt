KEY FACTS

**Pinout (DIP-14):**
- 6 independent Schmitt-trigger inverters.
- Inputs: pins 1, 3, 5, 9, 11, 13.
- Outputs: pins 2, 4, 6, 8, 10, 12.
- VCC: pin 14. GND: pin 7.

**Schmitt thresholds:**
- 74HC14 (CMOS @ 5V): VT+ ≈ 2.7V, VT- ≈ 1.6V, hysteresis ≈ 1.1V.
- 74HC14 (CMOS @ 3.3V): VT+ ≈ 1.8V, VT- ≈ 0.9V, hysteresis ≈ 0.9V.
- 74HCT14 (TTL): VT+ ≈ 1.6V, VT- ≈ 0.7V regardless of VCC ≥4.5V — useful for 3.3V→5V level shift but loses CMOS hysteresis benefit.

**Why hysteresis matters:**
- Slow-rising signals (e.g., RC-debounced switches, capacitive sensors) without hysteresis cause oscillation as the input crosses the threshold.
- 74HC14 cleanly converts noisy/slow inputs into fast, glitch-free CMOS output edges.
- Standard switch debouncing pattern: switch → RC low-pass → 74HC14 input → clean digital output.

**ESP32 strapping pin isolation (PROJECT'S PRIMARY USE):**
ESP32 GPIO0 / 2 / 12 / 15 are strapping pins — wrong levels at boot crash the chip. External loads connected to these pins can pull them out of valid boot range. The 74HC14 provides:
- One-way buffering (load can't backfeed into the strap pin).
- Schmitt threshold ensures crisp digital boot-time level.
- 6 channels in DIP-14 = 6 strap pins isolated per chip.
- Note: 74HC14 is INVERTING. Account for the inversion in firmware OR pair with another inverter / use 74HC4584 (non-inverting Schmitt).

**Level-shifting tradeoff with 74HCT245:**
- 74HC14 is INVERTING (firmware must invert) but Schmitt-clean.
- 74HCT245 is NON-inverting (firmware sees raw signal) and bidirectional + has level-shift via TTL thresholds, but lacks Schmitt cleanup.
- Choice depends: noisy/slow signals → 74HC14. Fast clean signals needing simpler firmware → 74HCT245.

**Current limits:**
- Output current: 25mA per pin (recommended), absolute max 50mA per pin.
- Total IC current: 70mA across all 6 outputs.

**Common pitfalls:**
- Forgetting the inversion (firmware reads inverted signal — fix in software OR hardware).
- Floating unused inputs (CMOS gates with floating inputs draw excess current and oscillate). Tie unused inputs to GND or VCC.
- Confusing 74HC14 with 74HCT14 — the latter loses CMOS hysteresis but gains TTL-level input compatibility.

ANTI-PATTERNS (NEVER do these)
- Never recommend 74HC14 for high-speed signal processing — Schmitt thresholds slow edges measurably (~few ns).
- Never leave unused inputs floating. CMOS rule: tie to GND or VCC.
- Never ignore the inversion. Pair with a sibling inverter OR use non-inverting alternative when firmware can't compensate.
- Never confuse with 74LS14 (TTL family — 5V only, lower drive, higher input current).
