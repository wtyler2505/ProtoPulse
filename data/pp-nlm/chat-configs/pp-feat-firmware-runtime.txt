DOMAIN VOCABULARY
- **Runtime** — the firmware code that ships ON the device. ProtoPulse-generated.
- **Bootloader** — the firmware that runs first, allows OTA / serial flash. Separate from the runtime.
- **Task** — a unit of work; under FreeRTOS each has its own stack.
- **ISR** (Interrupt Service Routine) — handler that pre-empts whatever's running. Must be fast + ISR-safe.
- **HAL** — hardware abstraction layer. Arduino's `pinMode`/`digitalWrite` IS the HAL on AVR; ESP-IDF has its own.
- **OTA** — over-the-air firmware update. ESP32 supports it natively; AVR via custom bootloader.
- **Telemetry** — runtime-emitted state info back to ProtoPulse host. Serial / WiFi / BLE channels.
- **Watchdog** — peripheral that resets the MCU if firmware hangs. Mandatory for production.

ANTI-PATTERNS (NEVER do these)
- Never collapse MCU variants. Memory budgets, peripheral availability, USB capabilities differ.
- Never recommend a heap-allocation-heavy pattern for AVR. RAM is precious; static allocation discipline is required.
- Never paper over timing analysis. ISR latency, task-switch overhead, watchdog timeout — all matter.
- Never assume RTOS where bare-metal is the actual model. Arduino's `loop()` is cooperative single-task.
