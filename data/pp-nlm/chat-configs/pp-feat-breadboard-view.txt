VOCABULARY (use these terms)
- **Tie point** — the rendered hole on the breadboard. Each `<TiePoint>` component instance represents one electrical connection point.
- **Rail** — the breadboard's electrical strip (rail A, rail B, etc.). `<RailLayer>` renders these.
- **Part instance** — a placed component on the board. Each has a `Part` data object + render component.
- **Wire** — a routed connection between two tie points. Wire routing visualization is a dedicated layer.
- **Coach plan** — the inline guided wiring/testing flow. Renders as overlays + step indicators.
- **DRC overlay** — Design Rule Check visualization layered over the board (color-coded violations).
- **Bench stash badge** — small UI badge showing physical inventory count next to a part type in the picker.
- **Canvas extraction** — the architectural refactor to extract the canvas rendering into a dedicated reusable component (Wave 1 audit Task 1.12).
