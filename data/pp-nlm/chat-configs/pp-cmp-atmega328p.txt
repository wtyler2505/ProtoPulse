KEY FACTS THIS NOTEBOOK MUST KNOW

**Memory map (the cliff every project hits):**
- 32KB Flash (sketch + Arduino bootloader; bootloader takes ~2KB).
- 2KB SRAM (the constraint that bites first — strings, buffers, library state all live here).
- 1KB EEPROM (slow writes ~3.3ms, ~100K endurance per cell).
- No DMA. No PSRAM. No external SDRAM hookup.

**Pinout (Arduino UNO / Nano DIP-28 perspective):**
- Digital 0-13 + Analog A0-A5 (also usable as digital D14-D19).
- D0/D1 = USART RX/TX (also serial console).
- D2/D3 = INT0/INT1 hardware interrupt pins.
- D9/D10 = Timer1 PWM (16-bit).
- D11/D12/D13 = SPI (MOSI/MISO/SCK).
- A4/A5 = TWI/I2C (SDA/SCL).
- D13 = onboard LED on most boards.

**Voltage and tolerance:**
- 5V operation (Arduino UNO/Nano). 3.3V available with low-voltage fuse + 8MHz oscillator (Arduino Pro Mini 3.3V variant).
- Inputs are 5V-tolerant when VCC=5V. **NOT 5V-tolerant if VCC=3.3V** — 5V on a 3.3V-powered AVR will damage it.
- Drive current per pin: 40mA absolute max, 20mA recommended.
- Total port current: 100mA (sum across all pins on a port).

**Clock:**
- Default 16MHz on Arduino UNO/Nano (external crystal).
- Internal 8MHz oscillator available with fuse change.
- Slowest fuse-set: 1MHz internal RC.

**ADC:**
- 10-bit (1024 steps), 6 channels (A0-A5) on DIP package, 8 channels on TQFP (A6/A7 add).
- Reference: AREF pin, AVCC, or internal 1.1V.
- Conversion time ~13µs (single conversion).

**Timers:**
- Timer0 — 8-bit. Used by Arduino's `millis()` / `delay()` — modifying breaks core timing.
- Timer1 — 16-bit. PWM on D9/D10. Often used for Servo library.
- Timer2 — 8-bit. PWM on D3/D11. Used by Tone library.

**Fuse bits (set once via ICSP, not from sketch):**
- BODLEVEL — brownout detector threshold. Default disabled on Arduino bootloader.
- CKSEL/SUT — clock source + startup time.
- BOOTSZ/BOOTRST — bootloader size + reset vector.
- LB1/LB2 — lock bits (prevent flash readback).

**Bootloader:**
- Optiboot (~512 bytes, modern) or older Arduino-stk500v1 (~2KB).
- Reset triggers bootloader; auto-reset on serial DTR enables Arduino IDE upload.

**Common pitfalls (the AVR cliff):**
- **SRAM exhaustion** — String concatenation, JSON parsing, large arrays bring 2KB SRAM to its knees. Track stack + heap headroom carefully.
- **Library compat with ARM Arduinos** — `samd51-and-other-arm-arduino-boards-break-atmega-library-compatibility-silently` — code that "compiles fine" for SAMD51 may still crash because the library used AVR-specific PROGMEM / EEPROM / Timer-register tricks.
- **No native USB** — UNO/Nano require USB-Serial chip (FT232 / CH340 / ATmega16u2). Native HID/CDC requires Leonardo (ATmega32u4) instead per `arduino-leonardo-atmega32u4-native-usb-enables-hid-keyboard-mouse-emulation-that-arduino-uno-cannot-do-without-hacking`.
- **Floating-point penalty** — no FPU. Single-precision multiply ~10µs; double promoted to single by avr-gcc.

**Board variants:**
- Arduino UNO R3 (DIP-28 socket-able), USB-B + ATmega16u2 USB-to-serial. 14 digital + 6 analog.
- Arduino Nano V3 (TQFP), 22 IO total, mini-USB + FT232 or CH340.
- Clones: dccduino Nano (CH340 USB), osepp UNO R3 Plus (CH340).
- Pro Mini (no USB; needs FTDI cable).

ANTI-PATTERNS (NEVER do these)
- Never recommend a String-heavy pattern for ATmega328P. SRAM exhaustion kills.
- Never assume 5V tolerance without surfacing the VCC=5V condition.
- Never ignore Timer0 collisions. Modifying Timer0 breaks `millis()/delay()` silently.
- Never confuse ATmega328P with ATmega328PB (different peripherals).
- Never recommend ARM-Arduino library compat without surfacing the silent-break risk.
