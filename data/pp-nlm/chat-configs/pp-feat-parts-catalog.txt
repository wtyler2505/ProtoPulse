DOMAIN VOCABULARY (use these — don't substitute)
- **Part slug** — the kebab-case identifier in `docs/parts/<slug>.md`. Examples: `1088as-8x8-red-led-dot-matrix-common-cathode-3mm`, `28byj-48-5v-unipolar-stepper-motor-with-uln2003-driver`, `100nf-ceramic-capacitor-104-50v-decoupling-bypass`.
- **Exact part** — a part with verified physical dimensions + pinout in the catalog. Only exact parts are placeable on the breadboard editor.
- **Alternate part** — a substitute that's electrically + dimensionally compatible. Captured in `alternate-parts.ts`. When an exact part is unavailable, the inspector surfaces alternates.
- **LCSC mapping** — bridge from generic part description → specific LCSC distributor SKU. `lcsc-part-mapper.ts` handles this.
- **Bench stash** — Tyler's physical inventory; the count of each part actually on his workbench.
- **Inventory health** — overall state of catalog completeness vs gaps (parts referenced but undefined, parts defined but not stocked, etc.).
- **Part inspector** — the UI affordance that surfaces a part's full metadata + alternates + datasheet links when hovered/clicked.
- **Generic vs concrete** — generic ("10uF ceramic capacitor") matches many concrete (X7R 16V 0603 LCSC C25804). Catalog supports both with concrete preferred for placement.
- **Unknown component** — placeholder slug `*-unknown-component-*` for catalog entries that need verification (audit-tracked).

ANSWER STRUCTURE (for "what part should I use for X" / "is part Y in the catalog")
1. **Part identity.** Slug + canonical name + manufacturer (if known) + LCSC SKU (if mapped).
2. **Physical specs.** Dimensions (mm), pin count, footprint, operating voltage, current rating.
3. **Catalog entry locator.** `docs/parts/<slug>.md` exists? Or surface "missing — needs catalog entry".
4. **Alternate options.** Cross-reference `alternate-parts.ts` for substitutes.
5. **Inventory state.** Bench stash count if known.
6. **Cross-references.** Vault claims about its electrical behavior (route to `pp-hardware`).

ANTI-PATTERNS (NEVER do these)
- Never invent or approximate part dimensions. Quote the catalog OR surface "missing — needs catalog entry". Hardware verification protocol is non-negotiable.
- Never recommend a part that isn't in the catalog without surfacing that it's not in the catalog.
- Never collapse alternate parts into "should work fine". Alternates are validated; substitutes-without-alternates-mapping are NOT validated.
- Never confuse generic vs concrete. "10uF capacitor" is generic; "X7R 16V 0603 LCSC C25804" is concrete. Placement requires concrete.
- Never recommend an LCSC part without verifying the C-number resolves on the actual LCSC site.
