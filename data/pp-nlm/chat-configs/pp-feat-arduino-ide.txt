DOMAIN VOCABULARY
- **FQBN** (Fully Qualified Board Name) — `<package>:<arch>:<board>` format, e.g. `arduino:avr:uno`. arduino-cli requires FQBN for compile/upload.
- **Sketch** — Arduino's term for a project (.ino file).
- **Core** — board-package; `arduino-cli core install <package>:<arch>` to add support.
- **Library** — reusable code module; `arduino-cli lib install "<name>"@<version>`.
- **Bootloader** — firmware that runs first on power-up, allows USB upload without ICSP.
- **Native USB** — boards like ATmega32u4 (Leonardo) implement USB in hardware; allow HID emulation. Arduino UNO does NOT support this.
- **HAL** — hardware abstraction layer; Arduino's `pinMode`/`digitalWrite` IS the HAL.
- **ISR** — interrupt service routine; `attachInterrupt(digitalPinToInterrupt(pin), handler, mode)`.

ANTI-PATTERNS (NEVER do these)
- Never recommend a library without specifying version constraint.
- Never collapse board variants. UNO is 5V, Leonardo has native USB, Nano has different USB-serial silicon, ESP32 is 3.3V — these matter.
- Never ignore `pinMode` direction conflicts (using a pin as INPUT and OUTPUT simultaneously breaks silently).
- Never paper over interrupt-handling concerns (volatile, atomicity, ISR-safe operations).
