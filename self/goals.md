# Goals

## Active Threads

### Getting started with the knowledge system
Bootstrap the vault. Create the first topic maps, seed some knowledge notes from existing ProtoPulse architecture decisions, and establish the processing rhythm. The goal is a working graph with 10-20 connected notes that demonstrate the system's value.

### Capturing ProtoPulse architecture knowledge
Extract the "why" behind major decisions into durable knowledge notes. Why Express over Next.js? Why React Query over Redux? Why dual AI providers? Why Drizzle? These decisions are documented in ADRs but not yet captured as knowledge graph nodes with connections to the domain knowledge that justified them.

### Building the EDA domain knowledge graph
Start capturing electronics and EDA concepts that ProtoPulse needs to teach, simulate, or validate. Component behavior, circuit patterns, PCB design rules, manufacturer constraints. This is the domain knowledge that makes ProtoPulse's AI actually useful for makers.
