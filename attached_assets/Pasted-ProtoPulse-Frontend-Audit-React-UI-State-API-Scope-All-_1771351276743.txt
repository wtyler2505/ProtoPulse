SECTION 2 — ARCHITECTURE & STRUCTURAL ISSUES
P1 Architecture — ChatPanel.tsx is 2,363 lines
Needs decomposition into ~6–8 components (e.g., MessageList, MessageInput, SettingsPanel, ModelSelector, AttachmentHandler, ImageGenPanel).

P1 Architecture/Perf — project-context.tsx is 614 lines with 40+ state values
Monolithic context triggers global re-renders; split into domain contexts (Chat/Diagram/BOM/Validation/etc.).

P1 Architecture — Sidebar.tsx is 832 lines
Should be section-driven subcomponents (navigation, tree, history, settings).

P1 Architecture — AssetManager.tsx is 678 lines
Mixing browsing/search/drag-drop/custom asset creation/shortcuts; split into dedicated modules.

P2 Performance — No code splitting / lazy loading
Heavy views (ReactFlow especially) are eagerly loaded; use React.lazy().

P1 Reliability — No tests anywhere
Zero unit/integration/e2e coverage for a state-heavy app.


SECTION 3 — STATE MANAGEMENT ISSUES
P0 Performance — Monolithic provider causes cascade re-renders
useProject() consumers rerender on any state change.

P2 Type Safety — setBomSettings accepts any (project-context line ~89)
Should be BomSettings or Partial<BomSettings>.

P1 Type Safety — ChatMessage.attachments?: any[] and actions?: any[]
Bypasses TS entirely; define interfaces.

P2 Type Safety — schematicSheets[].content: any
Define a schema/type for sheet content.

P0 Reliability — localStorage used as primary persistence for project state
~5MB limit, silent failure, not durable, no user feedback when exceeding.

P0 Reliability — No localStorage error handling
JSON.parse(localStorage.getItem(...)) with no try/catch; corrupted storage can brick app on load.

P1 Data Loss — Debounced saves can lose edits (ArchitectureView 1500ms debounce)
Closing tab within debounce window loses changes.

P1 Reliability — userInteracted ref flag is fragile
A shared boolean gate can be reset by unrelated updates; saves may skip incorrectly.

P0 Product — PROJECT_ID = 1 hardcoded
Blocks multi-project support and causes wrong-project assumptions.


SECTION 4 — TYPE SAFETY ISSUES
P2 Types — Widespread as any in CustomNode.tsx
nodes.find/map/filter((n: any) => ...) bypasses React Flow typing.

P2 Types — data.type as string, data.label as string assertions
Will explode silently if data shape changes.

P2 Types — ArchitectureView context menu uses any casts
Should use proper Node types.

P2 Types — iconMap typed as Record<string, any>
Should be Record<string, React.ComponentType<LucideProps>> (or similar).

P1 Correctness — Node IDs created via Date.now().toString()
Not guaranteed unique under fast creation; use crypto.randomUUID().

P0 Correctness — BOM deletion casts string ID to number
deleteBomItem(Number(item.id)) while interface defines id: string. Mismatch.

P0 Correctness — Validation deletion casts string ID to number
Same mismatch for validation issues.

P1 Correctness — addValidationIssue accepts loose severity: string
Should be 'error' | 'warning' | 'info'.


SECTION 5 — PERFORMANCE ISSUES
P2 Performance — SchematicView has massive hardcoded data objects (~220 LOC)
Extract to data files or fetch from backend.

P1 Performance — SchematicView re-renders entire SVG on panning
Mouse move triggers state updates → full SVG repaint; use ref + CSS transforms or canvas.

P2 Performance — No memoization on filtered BOM list
filteredBom recalculated each render; use useMemo.

P2 Performance — No memoization on filtered output log
filteredLog recalculated each render; use useMemo.

P1 Performance/Correctness — OutputView uses indexOf() per item
O(n²) and wrong index when duplicates exist.

P1 Performance — No virtualization for long lists
Output log, BOM, chat, validation render everything. Use react-window or @tanstack/react-virtual.

P2 Styling Fragility — ReactFlow controls styled with !important overrides
Fragile and likely to break on library updates.

P2 React — Multiple useCallback hooks with empty/incorrect deps
Risks stale closures or unnecessary rerenders.

P2 Perf/Tree Depth — Tooltip wrapping nearly every interactive element
Adds heavy component depth; consider fewer tooltips or a lighter pattern.
