### Categorization

The 54 dangling targets fall into 4 patterns, each requiring a different
remediation:

**Pattern A — Wiring guides (7 targets, ~103 refs).** Individual notes
reference named wiring guides (`wiring-36v-battery-power-distribution-
4-tier-system`, `wiring-zs-x11h-to-arduino-mega-for-single-motor-
control`, etc.) as if they exist. These are substantive composite
documents that need to be created, not bare links to fix. The refs are
spread between `knowledge/*.md` (foreign references from atomic notes)
and `ops/queue/wiring-*.md` (queue items where the filename's prefix
matches the dangling target — these are unprocessed extraction tasks
awaiting their parent wiring guide).

**Pattern B — Parts catalog (31 targets, ~100+ refs).** Module/IC catalog
entries like `l293d-dual-h-bridge-ic-...`, `28byj-48-5v-unipolar-
stepper-motor-...`, `pn2222a-npn-transistor-...`. The parts-
consolidation branch (current branch: `parts-consolidation`) appears to
be mid-flight. Many atomic notes reference these part cards but the
cards themselves haven't landed.

**Pattern C — `docs_and_data` source tag (1 target, 24 refs).** Used as
`Source: [[docs_and_data]]` in note footers. This is a canonical source
reference for notes extracted from the `docs_and_data.md` archive file.
Should be either:
- a topic map / source index file at `knowledge/docs_and_data.md`, or
- replaced with a direct `[[` link to the archived source file, or
- converted from `[[...]]` to plain text if not meant to be navigation.

**Pattern D — Corpus noise (~5 targets, ~13 refs).** `[[link]]`,
`[[links]]`, `[[wiki-links]]`, `[[power]]`. These are meta-references
inside methodology docs (`self/methodology.md`, `ops/methodology/
verify-wiki-links-before-completing-knowledge-work.md`) where the word
"link" happens to appear inside double brackets as an example or
placeholder, plus `[[power]]` in two knowledge notes that should
probably point to an existing `[[power-systems]]` topic map.

**Note on self-references:** the previous today's report
(`ops/health/2026-04-14-report.md`) itself contributes ~7 references to
the dangling list by naming the targets inside markdown link text. This
